Exploration Branch · Pretext for Diagram Layout

A bunch of examples, all focused on layout.

These examples keep our existing SVG-first diagram style and inject pretext only where text measurement changes the figure geometry. The goal is to show several different layout jobs we can hand off to the library without changing our broader rendering strategy.

Owns line wrapping, box height, shrink-wrap widths, label-mask sizing, responsive reflow
Does Not Own Mermaid internals, graph packing, ELK routing, or the editorial SVG rules themselves
Best Match inline SVG figures with long labels, mixed scripts, or multi-width rendering

Example 01

Measured node heights in a vertical architecture stack

Same editorial card language, but box height and connector anchors come from the label block instead of a guessed constant.

What Pretext is doing

  • Wrapping each label at a fixed content width.
  • Returning the honest line count for each node.
  • Driving the Y position of every downstream node.

Why it matters

If the second node grows by one line, the connector below it stays correct automatically.

Example 02

Flowchart boxes and the decision diamond sized from measured labels

The overall flow is still hand-placed. `pretext` just keeps each shape honest to its own content.

What Pretext is doing

  • Sizing the start pill and step boxes.
  • Increasing the diamond size when the decision text grows.
  • Keeping branch labels terse while the geometry expands.
Example 03

Edge-label masks that actually fit the label

Instead of a fixed 48px paper rectangle behind every arrow label, the mask width follows the real text width.

What Pretext is doing

  • Measuring mono edge labels.
  • Setting the white mask rectangle exactly once from the measured width.
  • Avoiding label bleed without leaving oversized blank slabs.
Example 04

Legend strip items shrink-wrapped to their content

This keeps the bottom legend from feeling arbitrary when one item is “OK” and the next is a full clause.

What Pretext is doing

  • Measuring each legend label individually.
  • Letting us place item groups with consistent gap instead of equal-width guesses.
  • Preserving the “legend as strip” rule without wasting space.

Takeaway

The useful adoption path is narrow and concrete.

We do not need to rebuild the diagram system around `pretext`. We only need one helper that measures text honestly and returns the geometry the figure code already needs: lines, line count, max width, box size, and anchor points.

  • Start with inline SVG node labels and edge-label masks.
  • Then use the same helper for legends and editorial callouts.
  • Keep Mermaid and manual graph layout exactly where they already belong.
