Diagram Lab · Optional Text Measurement Layer

Pretext makes the label box honest.

This exploration keeps our existing inline-SVG diagram style and only swaps in pretext at the text-measurement layer. The win is not global graph layout. The win is that node heights, edge anchors, and label masks stop being guessed.

What changes: line breaking, node height, shrink-wrap widths, connector anchors.

What stays: our diagram rules, 4px grid, typography, focal-accent discipline, Mermaid fallback.

Best fit: inline SVG diagrams with mixed scripts, dense labels, or responsive widths.

Panel B · Pretext

Measured line breaks

prepareWithSegments() caches the label and layoutWithLines() recomputes the wrapped lines against the current width. Boxes and anchors derive from that.

Where it helps

  • Inline SVG node labels with 2–5 lines.
  • Arrow-label masks sized from real text, not hard-coded rectangles.
  • Legends and annotation callouts that need honest widths.

Where it does not

  • It does not replace ELK, dagre, or any graph-packing step.
  • It does not buy much for a single short line inside a fixed box.
  • Mermaid still owns Mermaid. This is for our SVG-first path.

Integration shape

Build one cached helper that returns lines, maxLineWidth, boxWidth, boxHeight, and anchor points. Then let the figure layout code consume those numbers instead of hard-coded heights.

Recommended Wrapper

Minimal shape for a reusable SVG label helper.

const handle = prepareWithSegments(text, font);
const { lines } = layoutWithLines(handle, contentWidth, lineHeight);
const stats = measureLineStats(handle, contentWidth);

return {
  lines: lines.map((line) => line.text),
  lineCount: stats.lineCount,
  contentWidth: Math.ceil(stats.maxLineWidth),
  boxWidth: Math.ceil(stats.maxLineWidth + paddingX * 2),
  boxHeight: Math.ceil(stats.lineCount * lineHeight + paddingY * 2),
};

Official source: github.com/chenglou/pretext. Keep the prepared handle cache keyed by text + font + options, and prepare only after the font has loaded.
