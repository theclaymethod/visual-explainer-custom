Context · The problem Slide 03

We were losing telemetry during handler failures.

Whenever a handler panicked, its request/response was never logged. That blind spot made incident reconstruction unreliable and billing impossible to reconcile. The fix is to decouple logging from handling: write first, then route.

Owner · Platform Ref · RFC-204
Design Principles Slide 04

Three rules the ledger enforces.

Every component in the system derives its behavior from these three properties. Nothing downstream is allowed to violate them.

01
Write before handle. No handler runs until the request is durably logged.
02
Read from the ledger, not the stream. Downstream services subscribe by offset.
03
Replay is free. Any consumer can rewind to any offset at any time.

Interface · Gateway Slide 08

Producers call one function. Everything else is routing.

gateway/write.ts 
export async function writeRequest(req: Request): Promise<LedgerAck> {
  // Validate, then persist before any handler is notified.
  const record = await validate(req);
  const ack = await ledger.append(record, { quorum: true });
  // At this point the request is durable. Handlers read from ack.offset.
  return ack;
}
