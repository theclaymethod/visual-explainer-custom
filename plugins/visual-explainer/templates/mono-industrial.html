Request Ledger / Architecture v 2.4.0 2026·04·16

A write-first ledger for request and response telemetry.

Every inbound request is logged to append-only storage before any handler runs. Downstream services read from the ledger, never from the request stream directly.

01   Data Flow Mermaid / ELK

Write path, read path, and the durability boundary.

Producers write to the ledger via a thin gateway. Consumers subscribe by shard and replay from any offset. The gateway is the only component that can lose data; everything downstream is deterministic from the ledger tail.

02   Modules 4 services

Each module owns one side of the durability boundary.

Write · Svc
Gateway

HTTP ingress. Validates, rate-limits, then writes to the ledger. Returns 2xx only after durable ack from a quorum of ledger replicas.

Store · Core
Ledger

Append-only log, sharded by tenant. No reads from producers. Retention is configurable per shard; audit retention is pinned at 7 years.

Read · Svc
Indexer

Consumes the ledger tail and maintains secondary indexes. Cold-starts by full replay; recovers on its own after any failure.

Read · Svc
Query API

Serves admin UI and internal tools. Reads only from indexer state; never touches the ledger directly. No writes.
